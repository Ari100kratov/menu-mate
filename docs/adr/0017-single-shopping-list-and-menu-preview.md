# 0017 Единый список покупок и предпросмотр меню

## Статус

Принято.

## Контекст

Несколько сохраненных списков, отдельные состояния «куплено» и «есть дома», массовые режимы и мгновенная генерация из меню усложняли основной сценарий похода в магазин. Пользователю нужен один актуальный список, который можно дополнить вручную или полностью пересобрать из выбранного диапазона меню.

## Решение

- У владельца может быть не более одного списка покупок; уникальность обеспечивается индексом по `OwnerUserId`.
- Пустой список не создается заранее. Первая ручная позиция или подтвержденная генерация создают его лениво.
- Позиция имеет единственное состояние `IsPurchased`.
- Генерация из меню начинается с предпросмотра по блюдам. Пользователь меняет порции и выбирает ингредиенты.
- Подтверждение полностью заменяет текущие позиции и отметки.
- Backend повторно читает зафиксированные ревизии рецептов, масштабирует выбранные ингредиенты и объединяет совместимые позиции.
- Автоматически выполняются только детерминированные преобразования: кг в г и л в мл. Перевод бытовых объемных мер в массу требует коэффициента конкретного продукта и не выполняется универсально.

## Последствия

API и интерфейс работают с текущим списком без выбора идентификатора списка. Перезапись всегда требует явного предупреждения, а расчет нельзя подменить данными, измененными только на клиенте.
