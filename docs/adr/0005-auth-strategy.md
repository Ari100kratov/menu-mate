# 0005 Стратегия Auth

## Статус

Принято и реализовано для backend-среза.

## Контекст

Требования включают авторизацию, пользователей и будущие семейные пространства. Бэкенд больше не должен оставаться в режиме одного пользователя, потому что владение нужно до общих списков и пространств.

## Решение

Auth реализован как модуль внутри монолита:

- password hashing через PBKDF2;
- JWT access tokens;
- opaque refresh tokens в PostgreSQL;
- таблица ролей, сидируемая миграцией;
- persistence в схеме `auth`;
- `IUserContext` как общий способ получить текущего пользователя.

## Последствия

- Точка старта API настраивает middleware аутентификации и авторизации.
- Защищенные конечные точки используют bearer-токены.
- Семейные пространства остаются будущим расширением модели аутентификации и владения.
