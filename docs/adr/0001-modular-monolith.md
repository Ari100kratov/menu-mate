# 0001 Модульный монолит

## Статус

Принято.

## Контекст

MenuMate нужны понятные границы модулей, но на текущем этапе не нужна операционная сложность микросервисов.

## Решение

Начинаем как модульный монолит. Границы модулей выражаются проектами, namespaces, публичными contracts и владением persistence.

## Последствия

- Локальная разработка остается простой.
- На раннем этапе возможны локальные транзакции.
- Будущее выделение сервисов остается возможным, потому что границы явно видны в коде.
