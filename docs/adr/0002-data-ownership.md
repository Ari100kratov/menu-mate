# 0002 Владение данными модулей

## Статус

Принято.

## Контекст

Проекту нужен PostgreSQL и долгосрочная изоляция модулей без преждевременного ухода в микросервисы.

## Решение

Используем одну PostgreSQL базу с отдельной схемой и DbContext на stateful модуль. Отдельная база на модуль остается возможной позже, но не является дефолтом для MVP.

## Последствия

- Миграции сгруппированы по модулям.
- Прямой доступ к таблицам другого модуля запрещен.
- Межмодульные связи выражаются contracts, snapshots, projections и явными типами идентификаторов.
