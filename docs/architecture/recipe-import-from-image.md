# Создание рецепта из изображения

## Пользовательский поток

На странице рецептов кнопка добавления открывает меню с двумя сценариями: ручное создание и загрузка фото с ИИ. Сценарий импорта состоит из двух отдельных маршрутов:

- `/recipes/import` принимает от 1 до 8 скриншотов JPEG, PNG или WebP размером до 10 МБ каждый и до 40 МБ суммарно, а также показывает недавние черновики;
- `/recipes/import/:draftId` показывает все исходные скриншоты, предупреждения распознавания и обычную форму рецепта для проверки.

Изменения формы черновика сохраняются автоматически с задержкой. Рецепт создаётся только после явного нажатия «Создать рецепт». Исходные скриншоты не становятся обложкой автоматически. Черновик и исходники удаляются фоновым процессом через 7 дней после последнего изменения; срок настраивается через `RecipeImports:DraftRetentionDays`.

В формах создания, редактирования рецепта и проверки импортированного черновика рядом с загрузкой обложки доступна генерация фото блюда с ИИ. Генерация использует все текущие данные формы и возвращает квадратный JPEG `1024x1024` с низким качеством и сжатием, достаточным для карточки рецепта. Изображение остаётся локальным файлом формы и загружается как обычная обложка только при сохранении рецепта.

## Границы модулей

Модуль RecipeImports владеет черновиками, исходными изображениями и доказательствами распознавания. Он хранит данные в схеме PostgreSQL `imports`, а исходные изображения в закрытом бакете MinIO `imports`. Клиент получает только временную подписанную ссылку на исходник.

RecipeImports не записывает сущности Recipes напрямую. При подтверждении он вызывает команду создания Recipes с заранее выделенным идентификатором. Это делает повторное подтверждение идемпотентным при сбое между созданием рецепта и обновлением статуса черновика.

## Интеграция с OpenAI

Backend отправляет все изображения одним запросом в OpenAI Responses API и требует строгий JSON по схеме `CreateRecipeRequest` вместе с исходным распознанным текстом и предупреждениями. Модель по умолчанию: `gpt-5.4-mini`. Порядковые номера в начале распознанных шагов удаляются детерминированно после ответа модели.

Обложка создаётся через OpenAI Image API. Модель по умолчанию: `gpt-image-1-mini`; параметры изображения: качество `low`, размер `1024x1024`, формат JPEG, сжатие 75%.

Ключ задаётся только через `OpenAI:ApiKey` или `OPENAI_API_KEY`; хранить его в репозитории или передавать во frontend запрещено. Ответы модели не сохраняются на стороне OpenAI через параметр `StoredOutputEnabled = false`.

Доступные настройки:

- `OpenAI:Model` — модель распознавания рецепта;
- `OpenAI:ImageModel` — модель генерации обложки;
- `RecipeImports:DraftRetentionDays` — срок хранения черновика после последнего изменения;
- `RecipeImports:CleanupIntervalMinutes` — интервал фоновой очистки.

При прямом запуске API настройки видны в `src/MenuMate.Api/appsettings.Development.json`. При запуске через Aspire используются настройки из `src/MenuMate.AppHost/appsettings.Development.json`, после чего AppHost передаёт их API. Реальный ключ рекомендуется хранить в user secrets или переменной окружения, а поле `ApiKey` в репозитории оставлять пустым.

## API

Базовый маршрут: `/api/imports/recipe-drafts`.

- `POST /` создаёт черновик из multipart-коллекции `files`;
- `GET /` возвращает недавние черновики пользователя;
- `GET /{draftId}` возвращает черновик и временную ссылку на исходник;
- `PUT /{draftId}` заменяет редактируемое содержимое;
- `POST /{draftId}/confirm` создаёт рецепт;
- `DELETE /{draftId}` удаляет черновик и все исходные файлы.

Отдельный защищённый маршрут `POST /api/recipe-images/generate-cover` принимает `CreateRecipeRequest` и возвращает сгенерированный JPEG.
