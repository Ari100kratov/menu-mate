# Модули

## Auth

Владеет пользователями, ролями, refresh tokens и выпуском JWT.

Сейчас реализовано:

- регистрация и вход;
- refresh token rotation;
- конечная точка профиля текущего пользователя;
- logout через отзыв refresh tokens;
- схема PostgreSQL `auth`.

## Recipes

Владеет рецептами, ссылками на продукты каталога, количеством ингредиентов, шагами приготовления, категорией блюда, временем, флагом избранного, URL источника и snapshot тегов рецепта.

Сейчас реализовано:

- `Recipe`
- `RecipeIngredient`
- `IngredientQuantity`
- `RecipeTag`
- persistence в схеме `recipes`
- Recipes CRUD API
- масштабирование ингредиентов без изменения исходного рецепта
- владение через `UserId`

## Products

Владеет общим каталогом продуктов для рецептов и списков покупок.

Сейчас реализовано:

- нормализованное уникальное название продукта;
- категория продукта для группировки покупок;
- поиск для автодополнения;
- persistence в схеме `products`;
- разрешение нового названия в существующий или новый продукт.

## Tags

Владеет жизненным циклом тегов и нормализацией названий.

Сейчас реализовано:

- `Tag`
- `TagName`
- тип тега и статус подтверждения
- persistence в схеме `tags`
- Tags API

## MenuPlanning

Владеет планами меню и пунктами меню.

Сейчас реализовано:

- проверка диапазона дат;
- пункты меню на основе рецептов с snapshot названия рецепта;
- свободный текст без автоматического создания рецепта;
- количество порций на пункт меню;
- persistence в схеме `menu_planning`;
- MenuPlanning API;
- владение через `UserId`;
- связь с Recipes через `RecipeId`.

## ShoppingLists

Владеет генерацией, группировкой и текстовым представлением списка покупок.

Сейчас реализовано:

- перевод кг в г;
- перевод л в мл;
- осторожное объединение совместимых позиций;
- сохранение неопределенных количеств отдельными строками;
- группировка по категориям;
- persistence в схеме `shopping_lists`;
- ShoppingLists API;
- генерация из плана меню через `MenuPlanId`;
- ручное добавление, редактирование, удаление и отметки позиций;
- связь позиций с общим каталогом продуктов;
- объединение совместимых позиций по идентификатору продукта;
- текстовая версия списка для копирования.

## Cross-module recipe reuse

- Recipes owns authorship, visibility, user library entries, immutable revisions, and copy lineage.
- MenuPlanning stores both `RecipeId` and `RecipeRevisionId` for recipe-backed items.
- ShoppingLists reads pinned revision ingredients and never reads mutable current recipe ingredients.
- Recipe titles are presentation data and are intentionally not unique.

## Imports

Планируемый модуль для импорта из URL и изображений. Он должен хранить черновики, исходные доказательства извлечения и предложенные теги. Он не должен напрямую записывать данные в Recipes без подтверждения пользователя.

## Files

Планируемый модуль для загрузки и проверки файлов. Все изображения должны храниться в MinIO.

## Family Spaces

Планируемое расширение модели владения для общих книг рецептов, планов меню и списков покупок.
