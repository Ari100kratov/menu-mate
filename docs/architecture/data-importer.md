# Первоначальный импорт рецептов

## Назначение

`MenuMate.DataImporter` — отдельное консольное приложение для первоначального наполнения публичных рецептов. Оно не входит в Aspire AppHost и стандартный Docker Compose, потому что импорт является редкой административной операцией.

Схема состояния импорта принадлежит проекту `MenuMate.DataImporter.Infrastructure`. Ее миграции применяет стандартный `MenuMate.Migrator`.

Первичный русский справочник продуктов наполняется обычной миграцией EF Core в `ProductsDbContext`. Импортер рецептов не выполняет отдельный seed продуктов, но при создании рецептов существующая логика каталога может добавлять ингредиенты, которых еще нет в справочнике.

## Источник и повторяемость

- Рецепты загружаются из русской категории «Рецепты» Wikibooks через MediaWiki Action API.
- Рецепты находятся в специальном MediaWiki namespace `Рецепт` (`104`), поэтому страницы категории выбираются без ограничения namespace.
- Сначала используется детерминированный парсер разметки. Он поддерживает секции `Состав`/`Ингредиенты` и `Приготовление`, ингредиенты в форматах `количество продукт` и `продукт: количество`, а также первый вариант со свободным абзацем состава.
- Страницы с несколькими вариантами не смешиваются: выбирается первый полный вариант.
- Для книжных цитат, рецептов без явной структуры и других неподдерживаемых страниц применяется OpenAI Responses API.
- Каждая страница получает запись состояния с внешним идентификатором, номером редакции, хешем содержимого и заранее назначенным `RecipeId`.
- Повторный запуск не создает дубликаты. Неуспешные страницы можно повторно обработать с параметром `--resume`.
- Рецепты создаются через существующие application-команды от имени существующего пользователя с ролью `admin`.

Обложки загружаются только для изображений с лицензией CC BY, CC BY-SA или public domain. Файл копируется в MinIO, а URL источника, автор и лицензия сохраняются в metadata изображения и отображаются под ним.

## Запуск

Перед импортом нужно применить миграции. Текущая Auth-миграция создаёт bootstrap-администратора для первоначального наполнения. Его известные credentials предназначены только для локальной разработки: перед production-развёртыванием seed нужно заменить или удалить отдельной миграцией. Строка подключения, email администратора и ключ OpenAI не хранятся в tracked `appsettings.json` и передаются аргументами, user secrets или переменными окружения.

При запуске через Aspire миграции применяются автоматически. После старта AppHost скопируйте строку подключения ресурса `menumate` из Dashboard и передайте её импортеру сначала в dry-run:

```powershell
dotnet run --project src/MenuMate.DataImporter -- `
  --connection-string "Host=localhost;Port=<порт>;Database=menumate;Username=<пользователь>;Password=<пароль>" `
  --admin-email <email-bootstrap-администратора> `
  --dry-run `
  --max-items 20
```

При работе с базой данных, созданной Aspire, стандартный ресурс `migrator`
автоматически применяет в том числе миграции `MenuMate.DataImporter.Infrastructure`.
После добавления новой миграции достаточно перезапустить ресурс `migrator` или AppHost.
При обычном запуске импортер также самостоятельно применяет миграции принадлежащей
ему служебной схемы `data_import`. Режим `--dry-run` эту схему не читает и не изменяет.

`MenuMate.DataImporter` намеренно не входит в AppHost и не получает строку подключения автоматически. Aspire назначает host-port PostgreSQL динамически, поэтому адрес ресурса базы `menumate` нужно брать из текущего запуска Dashboard. Не используйте строку серверного ресурса `postgres`: в ней может отсутствовать `Database=menumate`, и Npgsql подключится к другой базе.

Вместо параметра можно задать переменную окружения:

```powershell
$env:ConnectionStrings__Database = "Host=localhost;Port=<порт>;Database=menumate;Username=<пользователь>;Password=<пароль>"
dotnet run --project src/MenuMate.DataImporter -- --admin-email <email-bootstrap-администратора> --dry-run --max-items 20
```

Поддерживаемые параметры:

- `--admin-email` — обязательный email существующего администратора;
- `--connection-string` — строка подключения к БД, например к динамической БД Aspire;
- `--dry-run` — проверка источника и парсинга без записи в БД и MinIO;
- `--resume` — продолжение ранее прерванного импорта;
- `--skip-images` — импорт рецептов без загрузки обложек;
- `--max-items` — ограничение количества страниц;
- `--source-category` — другая категория русского Wikibooks.

Настройки подключения, MediaWiki и OpenAI находятся в `src/MenuMate.DataImporter/appsettings.json` и могут быть переопределены переменными окружения.

После запуска рядом с рабочим каталогом создается JSON-отчет с количеством созданных и пропущенных рецептов, OpenAI fallback, изображений и ошибок.
